رفع المجلدات من الجوال

3 حلول مرتّبة من الأسهل — اختر واحداً

لماذا لا تستطيع رفع المجلدات؟
متصفح الجوال في وضعه العادي يرفع ملفات مفردة فقط، ولا يسمح باختيار مجلد كامل. هذه ميزة متاحة في وضع «الحاسوب». الحلول التالية تتجاوز هذه المشكلة.
⭐ الأسهل والموصى به

1) فعّل وضع «موقع الحاسوب» Desktop Site

حيلة بسيطة تجعل متصفح جوالك يرفع المجلدات تماماً كالحاسوب.

  1. افتح مستودعك على GitHub بمتصفح Chrome في جوالك.
  2. اضغط على النقاط الثلاث ⋮ أعلى يمين المتصفح.
  3. فعّل خيار «موقع كمبيوتر / Desktop site»:
    🔖 إضافة إلى المفضلة
    🖥️ موقع كمبيوتر ✓ 👈
    🔍 بحث في الصفحة
  4. الآن اضغط Add file → Upload files.
  5. عند الرفع، ستظهر إمكانية اختيار مجلد كامل — اختر مجلد ZonaSportApp.
  6. اضغط Commit changes. تم رفع كل شيء دفعة واحدة! ✅
💡 إن لم يظهر اختيار المجلد، جرّب الضغط مطوّلاً على منطقة الرفع، أو استخدم الحل الثاني.
📁 بديل بسيط جداً

2) ارفع ملف ZIP ثم استورده عبر «gitzip» أو يدوياً

GitHub لا يفكّ ZIP تلقائياً، لكن نتجاوز ذلك بإنشاء الملفات المهمة يدوياً.

  1. الملف الأهم الوحيد لتشغيل البناء هو ملف واحد فقط: .github/workflows/build-apk.yml
  2. اضغط Add file → Create new file.
  3. اكتب في خانة الاسم بالضبط:
    .github/workflows/build-apk.yml
    (الشرطة المائلة / تنشئ المجلدات تلقائياً!)
  4. الصق محتوى الملف، واحفظ (Commit).
  5. كرّر نفس الفكرة لبقية الملفات المهمة، أو ارفعها مجموعة بعد مجموعة عبر Upload files (ملفات لا مجلدات).
💡 خانة اسم الملف في GitHub هي الحيلة الذهبية: كل / تكتبها تنشئ مجلداً.
🔧 للمتقدمين

3) تطبيق MGit أو Termux (رفع كامل بأمر واحد)

يرفع المشروع كله مرة واحدة، لكنه يحتاج خطوات إعداد أكثر.

  1. ثبّت تطبيق MGit (أو Termux) من Google Play.
  2. أنشئ Personal Access Token من github.com/settings/tokens (صلاحية repo).
  3. في MGit: Clone مستودعك، انسخ ملفات المشروع داخله، ثم Add all → Commit → Push.
  4. يرفع كل المجلدات والملفات دفعة واحدة. ✅
💡 شرح Termux بالأوامر: pkg install git ثم git clone ثم git add . && git commit -m up && git push.
